Prólogo

Cuando la Web estaba en su infancia, hace unos cinco años, me sentí muy aliviado por la eliminación final de todos los problemas totalmente insolubles de la presentación en formato fijo. En la joven Web, ya no había más errores de paginación, ni más notas al pie, ni cortes de palabras tontos, ni ajustes del texto para ganar esa línea extra que tanto necesitabas para que todo cupiera en una página. En la ventana de una página web en el sistema NeXTStep, el texto siempre estaba limpio. Mejor aún: yo decidía en qué fuente aparecía, qué tan grandes quería las letras, qué estilos elegía para las listas de definiciones y dónde iban las tabulaciones.

Luego descendimos a la Edad Oscura durante varios años, porque la Web explotó en una comunidad que no tenía idea de que tal libertad era posible, pero se preocupaba por poner en la pantalla remota exactamente lo que pensaban que su información debería parecer. He leído recomendaciones en contra del uso de marcado estructurado porque no tienes control sobre lo que sale del otro lado. Triste.

A estas alturas habrás entendido que estoy firmemente en el campo de aquellos que piensan que la calidad del contenido viene primero, y la presentación viene después. Pero por supuesto, no estoy completamente en lo correcto aquí: la presentación es importante. Las fórmulas matemáticas siempre se presentan en un diseño bidimensional.

Afortunadamente, la filosofía de SGML nos permite separar la estructura de la presentación, y el DTD de la Web, HTML, no es una excepción. Incluso en la versión NeXTStep de 1990, Tim previó las hojas de estilo, aunque a un nivel rudimentario (¡teníamos otras cosas que hacer entonces!).

Hoy, las hojas de estilo están volviendo a ser una realidad, esta vez mucho más elaboradas. Este es un hito importante para la Web, y deberíamos detenernos un minuto para reflexionar sobre los beneficios potenciales y las trampas de esta tecnología.

Seguí el esfuerzo de CSS desde sus inicios - mayormente sobre tazas de café con Hakon en el CERN - y siempre tuve una preocupación: ¿es posible crear un "lenguaje" de hojas de estilo lo suficientemente poderoso sin terminar con un lenguaje de programación?

El CSS descrito en este libro muestra que puedes crear presentaciones bastante impresionantes sin programar. Aunque el programador en mí puede estar un poco decepcionado, el minimalista en mí está reconfortado. De hecho, nunca necesitaré tanta libertad y efectos especiales, pero entonces no soy un artista gráfico. Cualquier cosa que necesite más complicación se convierte efectivamente en una imagen, y debe ser tratada como tal. Siento, por lo tanto, que la parte media del espectro entre el texto ASCII puro y las imágenes completas está efectivamente cubierta por el poder de CSS, sin introducir la complejidad de la programación.

Tienes aquí un libro sobre presentación. Pero es presentación de información que también debería permanecer estructurada, para que tu contenido pueda ser utilizado efectivamente por otros, mientras retiene los aspectos visuales específicos que quieres darle. Usa CSS con cuidado. Es la sal largamente esperada en la comida Web: un poco es necesario, demasiado no es buena cocina.

Los esfuerzos de los autores finalmente nos han traído lo que tanto necesitábamos: la capacidad del autor para dar forma al contenido sin afectar la estructura. ¡Estas son buenas noticias para la Web!

Robert Cailliau
CERN, Ginebra
Febrero 1997

Prefacio

Este libro trata sobre una nueva forma de diseñar páginas web. Se llama Hojas de Estilo en Cascada (CSS) y revolucionará la manera en que los autores diseñan páginas web y cómo los usuarios las ven. CSS permite a los autores especificar cómo quieren que sus documentos aparezcan en pantallas o en papel, por ejemplo, qué fuentes y colores usar.

Desde que se escribió el primer documento web, los autores web han anhelado tener más control sobre sus documentos. Ese deseo ha sido tan fuerte que todos los medios posibles para influir en las pantallas de otras personas - hacer el texto en negrita, hacerlo parpadear, desplazarlo cinco píxeles a la derecha o desplazarse por la pantalla - han ganado seguidores. No importa cuán tonto sea el efecto, mientras los navegadores más populares lo soporten, se agrega a la lista cada vez mayor de extensiones HTML aceptadas. Al desarrollar CSS, nos aseguramos de ofrecer a la gente algunas nuevas herramientas interesantes - para satisfacer los deseos inmediatos. Además, pudimos introducir algunas características que creemos que realmente ayudarán a que la Web siga siendo un escenario de publicación para documentos más allá de los folletos de ventas.

Este libro está escrito para personas que diseñan páginas web. Aprenderás cómo CSS puede hacer tu trabajo más fácil y tus páginas lucir mejor. CSS ha sido creado para dar a los diseñadores web la influencia que desean mientras se mantiene la interoperabilidad y accesibilidad de la Web - y algunas otras "habilidades":

  • CSS ha sido desarrollado en un consorcio neutral de proveedores (el World Wide Web Consortium, dirigido por el mismo Tim Berners-Lee que inventó la Web y HTML).
  • CSS pronto llegará a un navegador cerca de ti.
  • CSS no rompe los documentos actuales, los aumenta.
  • CSS permite a los lectores, así como a los autores, definir hojas de estilo.
  • CSS utiliza terminología común de publicación de escritorio que probablemente ya conoces.
  • CSS permite hojas de estilo para toda la organización: solo necesitas cambiar un archivo cuando cambia el estilo de tu sitio.

Este libro te dirá todo lo que necesitas saber para comenzar a usar CSS.

Sitio web que acompaña este libro

En la Web, las cosas cambian rápidamente y CSS no es una excepción. Sin duda habrá nuevos desarrollos que informar en solo unos meses: más navegadores que soporten CSS, más características para usar en tus páginas web, más software para ayudarte a escribir CSS. Por esa razón, hemos creado un sitio web especial para mantenerte al día con los últimos desarrollos. Su dirección es http://www.awl.com/css/

El sitio web contiene actualizaciones del texto del libro, así como muchos hipervínculos adicionales relacionados con CSS. También puedes encontrar allí todos los ejemplos del libro, listos para ser cortados y pegados en tus propias páginas. Y si hubiera alguna errata en el libro, también se publicará allí.
