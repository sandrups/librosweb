Uso de JavaScript: Un Lenguaje de Scripting Simple y Basado en Objetos

A diferencia de Java, JavaScript se une a las filas de los lenguajes de scripting simples y fáciles de usar. JavaScript promete una curva de aprendizaje más fácil que Java junto con herramientas poderosas para agregar interactividad a las páginas web con poco esfuerzo.

Derivado de Java

JavaScript debe mucho a Java. Su sintaxis y estructura básica son similares a las de Java, incluso si el rango de funciones y el estilo de programación pueden diferir enormemente. JavaScript comenzó su vida como el propio lenguaje de scripting de Netscape con el nombre LiveScript, pero a finales de 1995, Sun respaldó el lenguaje, y se convirtió en JavaScript.

JavaScript conserva más que solo la sintaxis y estructura básica de Java: también toma prestadas la mayoría de las construcciones de flujo de Java e implementa algunas de las mismas precauciones de seguridad, como prevenir que los applets escriban en el disco local.

Un Lenguaje Interpretado

A diferencia de Java, JavaScript es un lenguaje interpretado. Mientras que en Java, el código fuente se compila antes de la ejecución, en un lenguaje interpretado, los archivos de código fuente se ejecutan directamente en tiempo de ejecución en JavaScript.

Los lenguajes interpretados ofrecen varias ventajas, así como varios inconvenientes. Los lenguajes interpretados, como JavaScript, generalmente son más simples que los lenguajes compilados y son fáciles de aprender. A menudo es más fácil desarrollar, cambiar y depurar programas porque se elimina la necesidad de recompilar con cada cambio.

En el lado negativo, la necesidad de interpretar comandos mientras se ejecuta el programa puede generar una disminución en el rendimiento con algunos lenguajes interpretados. En el caso de JavaScript, esto no parece ser un problema.

Los scripts de JavaScript se compilan en códigos de bytes (similar a Java) a medida que se descargan y evalúan, y para la mayoría de los scripts, el rendimiento es bastante rápido.

No totalmente extensible

A diferencia de Java, JavaScript no es totalmente extensible. El modelo de JavaScript es un conjunto limitado de objetos base, propiedades, métodos y tipos de datos, que proporcionan suficientes capacidades para crear aplicaciones del lado del cliente o del lado del servidor.

Aunque los usuarios pueden crear sus propios objetos y escribir funciones, esto no es lo mismo que las clases y la herencia que ofrecen Java y otros lenguajes de programación orientados a objetos.

Dado que JavaScript es un lenguaje de scripting basado en objetos, este libro estará dedicado a aprender sobre los objetos, propiedades, métodos y tipos de datos disponibles en JavaScript.

Interacción limitada cliente-servidor

JavaScript en su forma actual no está diseñado para una interacción completa cliente-servidor. Más allá de analizar, construir e invocar URL, JavaScript no puede comunicarse directamente con los servidores ni utilizar diferentes protocolos. Esencialmente, JavaScript es adecuado para manejar la actividad del lado del cliente.

Aun así, hay indicios de que las futuras versiones de JavaScript soportarán protocolos como HTTP y FTP, pero varios problemas de seguridad relacionados con la interacción cliente-servidor a través de JavaScript deben ser abordados por Netscape antes de implementar estas características.

Además, JavaScript es parte del servidor web LiveWire ofrecido por Netscape. LiveWire está dirigido a grupos y organizaciones que desarrollan aplicaciones web interactivas. En este rol, JavaScript proporciona una alternativa al scripting CGI tradicional para la programación del lado del servidor. En este sentido, JavaScript puede usarse para la interacción cliente-servidor.

Integrado en HTML

Mientras que Java solo está ligeramente vinculado a HTML, JavaScript está estrechamente integrado en los archivos HTML. Por lo general, los scripts completos se encuentran en los mismos archivos que el HTML que define una página, y estos scripts se descargan al mismo tiempo que los archivos HTML.

El estado actual de JavaScript

Al emprender la tarea de aprender JavaScript, es importante tener en cuenta el estado actual del lenguaje y hacia dónde parece dirigirse su desarrollo.

En desarrollo

Tanto Navigator 3.0 como JavaScript están en un desarrollo intenso y están evolucionando constantemente. Entonces, JavaScript es, por definición, un lenguaje en desarrollo (como lo son la mayoría de los lenguajes de programación). En términos prácticos, esto significa que la especificación completa del lenguaje aún no se ha implementado.

De una manera muy real, JavaScript es un objetivo en movimiento. De Navigator 2.0 a Navigator 3.0, el conjunto de características se ha ampliado enormemente y ha cambiado notablemente. Además, algunos métodos y propiedades existentes han sido modificados. No hay razón para creer que este desarrollo y cambio no se extenderá a futuras versiones de Navigator.

Además de esto, la implementación de JavaScript por parte de Microsoft difiere sutilmente de la de Netscape, y no hay garantía de que estas diferencias no persistan o cambien con el tiempo.

Apoyado por Sun

A finales de noviembre de 1995, LiveScript se convirtió en JavaScript en un anuncio conjunto de Netscape y Sun, en el que Sun anunció que apoyaría JavaScript como el estándar de scripting abierto basado en Java para Internet.

Dado este apoyo, hay pocas dudas de que JavaScript continuará pareciéndose a Java y que se convertirá en la herramienta elegida para integrar applets de Java en las páginas web, al menos en el entorno de Netscape.

Esencialmente, este respaldo de Sun le dio vida a un lenguaje de scripting al que muy pocas personas prestaban atención, pero que ahora es el tema candente en la World Wide Web.

Respaldado por muchas empresas

Sun Microsystems no es la única empresa que apoya JavaScript. Al mismo tiempo que Netscape y Sun hicieron su anuncio, más de 28 empresas, incluidas America Online, Apple Computers, Oracle, Silicon Graphics, Architext y SCO, anunciaron que también respaldarían JavaScript como el estándar de scripting abierto para Internet, y muchas indicaron que estaban considerando licenciar la tecnología para incluirla en sus propios productos.

Uso Potencial en Diferentes Productos

Además de incluir JavaScript en su navegador web Navigator, Netscape está incluyendo JavaScript en su entorno de desarrollo de aplicaciones del lado del servidor llamado LiveWire.

Esto hace posible que el impulso para la programación CGI utilizando Perl y Bourne o C shell pueda trasladarse hacia JavaScript, lo que proporcionaría un entorno de desarrollo consistente tanto en el cliente como en el servidor.

Además, los respaldos de numerosas empresas significan que JavaScript comenzará a aparecer en productos de compañías distintas a Netscape y en productos diferentes a navegadores y servidores web. Como se mencionó anteriormente, JavaScript ya es compatible con el Internet Explorer de Microsoft.

Además, Netscape ha propuesto JavaScript como un estándar de scripting para la World Wide Web.

JavaScript Hoy: Scripting para el Navegador Web de Netscape

En este libro, nos estamos enfocando en el uso de JavaScript en el navegador web Navigator porque es donde se está llevando a cabo la mayor parte del desarrollo de JavaScript y porque es el entorno más fácil en el que aprender y practicar JavaScript. JavaScript también está disponible en la nueva versión 3 de Internet Explorer de Microsoft. Internet Explorer también incluye otro lenguaje de scripting, VBScript, basado en el lenguaje de programación Visual Basic de la compañía. VBScript no se cubre en este libro.

¿Qué es JavaScript?

A estas alturas, probablemente tengas una idea de lo que es JavaScript y lo que no es, pero veamos más de cerca las características que aprenderás y usarás a lo largo de este libro.

JavaScript es un Lenguaje de Scripting

Los lenguajes de scripting se han utilizado mucho antes de la llegada de la Web. En el entorno UNIX, los scripts se han utilizado para realizar tareas repetitivas de administración del sistema y para automatizar muchas tareas para usuarios menos expertos en informática. Además, los lenguajes de scripting son la base de gran parte de la programación CGI-BIN que se utiliza actualmente para agregar una forma limitada de interactividad a las páginas web.

Ejemplos de lenguajes de scripting incluyen Perl, bien conocido en la programación CGI, Awk y SED (diseñados para procesamiento intensivo de texto), e incluso HyperTalk que, al igual que JavaScript, es un lenguaje de scripting orientado a objetos.

Por supuesto, esto aún no te dice cuáles son las principales ventajas de los lenguajes de scripting. Como todos los lenguajes de scripting, JavaScript es interpretado, lo que proporciona un proceso de desarrollo fácil; contiene un conjunto de comandos y una sintaxis limitados y fáciles de aprender; y está diseñado para realizar un conjunto bien definido de tareas.

Diseñado para Programas Simples y Pequeños

Debido a que JavaScript es un lenguaje de scripting, está bien adaptado para implementar programas simples y pequeños. Por ejemplo, JavaScript sería ideal para desarrollar una calculadora de conversión de unidades entre millas y kilómetros o libras y kilogramos. Estas tareas se pueden escribir y realizar fácilmente a velocidades aceptables con JavaScript y se integrarían fácilmente en una página web. Un lenguaje más robusto como Java sería mucho menos adecuado para el desarrollo rápido y el fácil mantenimiento de este tipo de aplicaciones.

Por el contrario, JavaScript no sería adecuado para implementar un entorno distribuido de visualización y manipulación de documentos CAD. Aunque eventualmente JavaScript será una herramienta para integrar este tipo de applet o complemento de Java en una página web, intentar desarrollar el applet real en JavaScript sería, en el mejor de los casos, difícil e ineficiente y, más probablemente, imposible.

Por supuesto, esto no significa que no se puedan desarrollar aplicaciones sofisticadas con JavaScript, y de hecho, se están desarrollando. No obstante, los lenguajes de scripting generalmente se usan para tareas más pequeñas en lugar de para programas completos compilados.

Realiza Tareas Repetitivas

Así como JavaScript es adecuado para producir programas pequeños, está especialmente bien diseñado para tareas repetitivas invocadas por eventos. Por ejemplo, JavaScript es ideal para calcular el contenido de un campo en un formulario basado en cambios en los datos de otro campo. Cada vez que los datos cambian, se invoca el programa de JavaScript para manejar el evento, y los nuevos datos para el otro campo se calculan y se muestran.

Diseñado para Programar Eventos de Usuario

Debido a la forma en que JavaScript está integrado en el navegador y puede interactuar directamente con las páginas HTML, JavaScript hace posible programar respuestas a eventos de usuario como clics de ratón y la entrada de datos en formularios.

Por ejemplo, se podría usar un script de JavaScript para implementar un sistema de ayuda simple. Cada vez que el usuario señale un botón o un enlace en la página, se puede mostrar un mensaje útil e informativo en la barra de estado en la parte inferior de la ventana del navegador.

Esto añade interactividad a las páginas web, hace que los formularios sean dinámicos y puede reducir los requisitos de ancho de banda y la carga del servidor que conlleva el uso de formularios y la programación CGI.

Fácil Depuración y Pruebas

Al igual que otros lenguajes de scripting, JavaScript facilita el desarrollo y la resolución de problemas porque no está compilado. Es fácil probar el código del programa, ver los resultados, hacer cambios y probarlo nuevamente sin la sobrecarga y el retraso de la compilación.

El Pegamento de Java

Cuando Netscape anunció JavaScript, se refirió al lenguaje como la herramienta para "pegar applets de Java en páginas web". Con Navigator 3.0 esto se ha vuelto posible. JavaScript puede desencadenar eventos en applets de Java, y los applets de Java pueden llamar a métodos y funciones de JavaScript.

JavaScript es Basado en Objetos

Orientado a objetos es un término que ha sido sobreutilizado por los medios y el brazo de marketing de las industrias de computación y software. No obstante, el hecho de que JavaScript tenga un modelo orientado a objetos limitado es una distinción importante.

El Modelo de Objetos de JavaScript

Para entender qué significa que JavaScript sea basado en objetos, necesitamos observar los objetos y cómo funcionan.

Fundamentalmente, los objetos son una forma de organizar información, junto con los métodos para manipular y usar esa información.

Los objetos proporcionan una manera de definir piezas específicas de datos relacionadas con el ítem en cuestión; estas piezas se conocen como propiedades. Además, estas se complementan con tareas que se pueden realizar con o sobre esa información, conocidas como métodos. Juntos, las propiedades y los métodos conforman los objetos.

Debido a la naturaleza general de los objetos, se pueden crear instancias específicas para cada caso donde se necesiten. Por ejemplo, un objeto coche podría tener varias instancias para Toyotas, Fords y Volkswagens.

Una Comparación con los Lenguajes Procedurales

Definir objetos de esta manera difiere mucho de la forma en que se maneja la información en los lenguajes de programación procedurales tradicionales como FORTRAN y C.

En estos lenguajes, la información y los procedimientos (similares a los métodos) se mantienen separados y distintos y no están vinculados de la manera en que lo están los objetos. Además, el concepto de crear instancias no está tan desarrollado en los lenguajes procedurales.

Trabajando con Objetos en JavaScript

JavaScript incluye objetos incorporados para trabajar con elementos del documento HTML actualmente cargado, así como para realizar otras tareas útiles, como cálculos matemáticos. También ofrece al programador la oportunidad de crear sus propios objetos.

Objetos Incorporados

JavaScript ofrece un conjunto de objetos incorporados que proporcionan información sobre la página web actualmente cargada y su contenido, así como sobre la sesión actual del Navegador. Además, estos objetos proporcionan métodos para trabajar con sus propiedades.

Extendiendo JavaScript: Creando tus propios objetos

Además de una amplia gama de objetos integrados, como programador de JavaScript, puedes crear tus propios objetos para usar en tus scripts.

Propiedades

Por ejemplo, si necesitas construir un objeto en JavaScript para representar los diferentes tipos de aviones vendidos por un fabricante de aeronaves, tendrías varios datos relacionados con el avión que querrías incluir en el objeto, incluyendo los siguientes:

  • Modelo
  • Precio
  • Capacidad de asientos normal
  • Capacidad de carga normal
  • Velocidad máxima
  • Capacidad de combustible

Las propiedades son como variables en lenguajes tradicionales, como C y Pascal. Las variables son contenedores nombrados que se utilizan para almacenar datos, como números o texto. Las variables se discuten con más detalle en el Capítulo 3, "Trabajando con Datos e Información".

Entonces, en JavaScript, si llamas a tu objeto avión, estas propiedades podrían referirse como:

    avión.modelo
    avión.precio
    avión.asientos
    avión.carga
    avión.velocidadmax
    avión.combustible
Nota
Como aprenderás más adelante, en JavaScript, las propiedades de los objetos se refieren mediante la estructura nombre-del-objeto.nombre-de-la-propiedad.
Métodos

Por supuesto, tener esta información no vale mucho sin formas de usarla. Por ejemplo, en este ejemplo, quieres poder imprimir una descripción bien formateada de la aeronave o ser capaz de calcular la distancia máxima que el avión puede recorrer según la capacidad de combustible.

En la terminología orientada a objetos, estas tareas se conocen como métodos. Al igual que las propiedades, tus métodos podrían ser referidos como:

    avion.descripcion()
    avion.distancia()
Objetos dentro de Objetos

Los objetos también pueden incluir otros objetos de manera similar a las propiedades y métodos. Por ejemplo, el fabricante de aviones puede querer incluir un objeto dentro de su definición de objeto para manejar información sobre el número de aviones en uso en todo el mundo, quién los está usando y sus registros de seguridad. Esta información, junto con métodos para trabajar con la información, podría combinarse en un objeto llamado avion.registro. Este objeto luego podría incluir propiedades y métodos como:

    avion.registro.numero_en_uso
    avion.registro.accidentes
    avion.registro.nuevotamaño()
Instancias

Lo que has creado es una descripción general de un objeto que define la información con la que quieres trabajar y las formas en que deseas trabajar con ella. Estas descripciones generales de objetos se conocen como clases. En la programación orientada a objetos, entonces, puedes crear instancias específicas de la clase según sea necesario.

Nota:
En la programación orientada a objetos, crear copias específicas de clases se conoce como crear instancias. JavaScript en sí mismo es sin clases, pero proporciona mecanismos para crear instancias de objetos (proporcionando así la misma funcionalidad básica). Cubrimos los detalles de la creación de objetos e instancias en el Capítulo 4, "Funciones y Objetos: Los componentes básicos de los programas".

Por ejemplo, en este ejemplo, el fabricante de aviones podría querer crear una instancia del objeto avión para su nueva aeronave, el SuperAvión. Si se creara esta instancia, entonces un programa podría asignar valores específicos a las propiedades de la nueva instancia refiriéndose a superavion.precio, superavion.modelo, y así sucesivamente. Del mismo modo, una descripción del nuevo avión podría imprimirse utilizando superavion.descripcion().

Fortalezas de JavaScript

JavaScript ofrece varias fortalezas al programador, incluyendo un ciclo de desarrollo corto, facilidad de aprendizaje y scripts de tamaño reducido. Estas fortalezas significan que JavaScript puede ser utilizado fácil y rápidamente para extender páginas HTML ya existentes en la Web.

Desarrollo Rápido

Dado que JavaScript no requiere una compilación que consume mucho tiempo, los scripts pueden ser desarrollados en un período relativamente corto. Esto se ve favorecido por el hecho de que la mayoría de las características de la interfaz, como cuadros de diálogo, formularios y otros elementos de GUI, son manejados por el navegador y el código HTML. Los programadores de JavaScript no tienen que preocuparse por crear o manejar estos elementos de sus aplicaciones.

Fácil de Aprender

Aunque JavaScript puede compartir muchas similitudes con Java, no incluye la sintaxis compleja ni las reglas de Java. Al aprender solo unos pocos comandos y reglas simples de sintaxis, junto con entender la manera en que se utilizan los objetos en JavaScript, es posible comenzar a crear programas bastante sofisticados.

Independencia de Plataforma

Debido a que la World Wide Web, por su propia naturaleza, es independiente de la plataforma, los programas de JavaScript creados para Netscape Navigator no están ligados a ninguna plataforma de hardware específica o sistema operativo. El mismo código de programa puede ser utilizado en cualquier plataforma para la cual esté disponible Navigator 2.0.

Bajo Sobrecosto

Los programas de JavaScript tienden a ser bastante compactos y son bastante pequeños, en comparación con los applets binarios producidos por Java. Esto minimiza los requisitos de almacenamiento en el servidor y los tiempos de descarga para el usuario. Además, debido a que los programas de JavaScript generalmente se incluyen en el mismo archivo que el código HTML de una página, requieren menos accesos a la red por separado.

Debilidades de JavaScript

Como era de esperarse, JavaScript también tiene sus propias debilidades únicas. Estas incluyen un conjunto limitado de métodos incorporados, la incapacidad de proteger el código fuente de miradas curiosas, y el hecho de que JavaScript aún no tiene un entorno maduro de desarrollo y depuración.

Rango Limitado de Métodos Incorporados

Las primeras versiones beta de Navigator 2.0 incluían una versión de JavaScript que era bastante limitada. En la versión final de Navigator 2, el número de métodos incorporados había aumentado significativamente, pero aún no incluía un conjunto completo de métodos para trabajar con documentos y las ventanas del cliente. Con el lanzamiento de Navigator 3, las cosas han avanzado un paso más con la adición de numerosos métodos, propiedades y manejadores de eventos.

Falta de Ocultación de Código

Debido a que el código fuente de los scripts de JavaScript actualmente debe incluirse como parte del código fuente HTML de un documento, no hay forma de proteger el código de ser copiado y reutilizado por las personas que ven tus páginas web.

Esto plantea preocupaciones en la industria del software sobre la protección de la propiedad intelectual. El consenso es que los scripts de JavaScript son básicamente freeware en este momento.

Falta de Herramientas de Depuración y Desarrollo

La mayoría de los entornos de programación bien desarrollados incluyen una suite de herramientas que facilitan el desarrollo y simplifican y aceleran el proceso de depuración.

Actualmente, existen algunos editores HTML y editores de programación que brindan soporte para JavaScript. Además, hay algunas herramientas en línea para depurar y probar scripts de JavaScript. Sin embargo, realmente no existen entornos de desarrollo integrados como los disponibles para Java, C o C++. LiveWire de Netscape proporciona algunas características de desarrollo, pero no es un entorno de desarrollo completo para JavaScript del lado del cliente.

Resumen

JavaScript es uno de varios componentes clave en Netscape Navigator diseñados para ayudar a los desarrolladores web a producir aplicaciones interactivas en Internet. Otras características incluyen soporte para plug-ins, frames y, por supuesto, Java.

Donde Java es compilado, orientado a objetos, complejo y distinto de HTML, JavaScript es interpretado, basado en objetos, sencillo de usar y aprender, e íntimamente integrado con HTML.

JavaScript expone propiedades de la sesión actual del navegador al programador, incluyendo elementos de las páginas HTML cargadas actualmente, tales como formularios, marcos y enlaces.

Incorporando JavaScript en HTML

En la actualidad, todos los scripts de JavaScript deben ser incluidos como parte integral de un documento HTML. Para hacer esto, Netscape ha implementado una extensión al HTML estándar: la etiqueta SCRIPT.

La etiqueta script

Incluir scripts en HTML es simple. Cada script debe estar contenido dentro de una etiqueta contenedora SCRIPT. En otras palabras, una etiqueta de apertura <SCRIPT> inicia el script y una etiqueta de cierre </SCRIPT> lo termina:

    
      <SCRIPT>
      Programa en JavaScript
      </SCRIPT>

La etiqueta script tiene dos atributos opcionales que determinan cómo se incorpora el script de JavaScript en cuestión al archivo HTML. Estos atributos se describen en la Tabla 2.1.

Tabla 2.1. Atributos para la etiqueta script.
Atributo Descripción 
SRC
 URL de un archivo que contiene el código fuente de JavaScript. El archivo debe tener la extensión .js. Consulta el sitio web de Navigator para más detalles sobre la etiqueta SRC. 
LANGUAGE
 Indica el idioma utilizado en el script. En la versión actual de Navigator, este atributo puede tomar solo dos valores: JavaScript y LiveScript. LiveScript se proporciona para la compatibilidad con versiones anteriores de scripts desarrollados cuando el idioma se llamaba LiveScript. Debes usar JavaScript en tus scripts.
Nota:
Usar la etiqueta SCRIPT y sus atributos te permite usar dos técnicas para integrar un programa JavaScript en un archivo HTML. Sin embargo, en Navigator, los programadores solo tienen una opción: incluir sus programas JavaScript en sus archivos HTML.
Consejo:
Aunque Navigator admite el uso de LANGUAGE="LiveScript" para la compatibilidad con versiones anteriores, esto se eliminará en futuras versiones del navegador. Es mejor usar LANGUAGE="JavaScript". Además, LANGUAGE="LiveScript" no es compatible con Internet Explorer 3.0 de Microsoft.

Incluir JavaScript en un archivo HTML

La primera, y más fácil, manera es incluir el código fuente real en el archivo HTML, usando la siguiente sintaxis:

    
      <SCRIPT LANGUAGE="JavaScript">
      programa JavaScript
      </SCRIPT>

Ocultar scripts de otros navegadores

Por supuesto, surge un problema inmediato con este tipo de contenedor SCRIPT: Los navegadores que no admiten JavaScript intentarán mostrar o analizar el contenido del script. Para evitar esto, Netscape recomienda el siguiente enfoque utilizando comentarios HTML:

    
      <SCRIPT LANGUAGE="JavaScript">
      <!-- OCULTAR EL SCRIPT DE OTROS NAVEGADORES
      Programa en JavaScript
      // DEJAR DE OCULTAR DE OTROS NAVEGADORES -->
      </SCRIPT>

Estos comentarios (<!-- OCULTAR EL SCRIPT DE OTROS NAVEGADORES y // DEJAR DE OCULTAR DE OTROS NAVEGADORES -->) aseguran que otros navegadores web ignorarán todo el script y no lo mostrarán, ya que todo lo que está entre <!-- y --> debería ser ignorado por un navegador web estándar. Por supuesto, si los usuarios vieran el código fuente del documento, aún verían el script.

Problemas con la etiqueta script

Esta técnica de combinar la etiqueta SCRIPT con comentarios no es infalible, sin embargo. En este momento, la etiqueta SCRIPT no es una parte aceptada del estándar HTML 2.0 y la especificación HTML 3.0 está incompleta. Por el momento, los fabricantes de navegadores competidores podrían usar la etiqueta SCRIPT para otro propósito.

Nota:
En el momento de la publicación, parecía que la etiqueta SCRIPT, y posiblemente JavaScript en sí mismo, se convertiría en parte de la especificación HTML 3.0 porque cuenta con el respaldo de Netscape y Sun, entre otros.

De hecho, con Netscape Navigator 2.0, este último problema ya ha ocurrido con la implementación de marcos. Entre varias etiquetas usadas para producir marcos, Netscape usa una etiqueta FRAME que es utilizada por el Web Explorer de IBM para otro propósito.

Además, al ocultar el script de otros navegadores, los usuarios de estos otros navegadores no serán conscientes de la existencia del script. Una solución a este problema podría ser usar el siguiente enfoque: 

    
      <SCRIPT LANGUAGE="JavaScript">
      // El Script de JavaScript Aparece Aquí<BR>
      // Descargue Netscape Navigator para usarlo.
      <!-- OCULTANDO DE OTROS NAVEGADORES
      Programa en JavaScript
      // DEJAR DE OCULTAR DE OTROS NAVEGADORES -->
      </SCRIPT>
Nota: A diferencia del HTML, que crea comentarios con la estructura <!-- Comentarios Aquí -->, los comentarios en JavaScript comienzan con una doble barra (//) en cualquier parte de la línea y continúan hasta el final de la línea. Si no están contenidos dentro de una estructura de comentario HTML, un comentario de JavaScript será mostrado por navegadores que no sean Netscape. Como aprenderás más adelante en este capítulo, JavaScript también incluye comentarios de múltiples líneas.

A partir de Navigator 3, Netscape ha introducido el elemento NOSCRIPT, que proporciona una forma de especificar texto alternativo para navegadores que no soportan JavaScript. Cualquier texto entre las etiquetas NOSCRIPT será mostrado por otros navegadores, pero Navigator 3 lo ignorará. El ejemplo anterior podría implementarse usando la etiqueta NOSCRIPT:

    
      <SCRIPT LANGUAGE="JavaScript">
      <!-- ESCONDIÉNDOLO DE OTROS NAVEGADORES
      Programa en JavaScript
      // DEJAR DE ESCONDERLO DE OTROS NAVEGADORES - ->
      </SCRIPT>
      <NOSCRIPT>
      El script de JavaScript aparece aquí<BR>
      Descarga Netscape Navigator 2.0 para usarlo.
      </NOSCRIPT>

Los scripts y programas JavaScript se pueden incluir en cualquier lugar del encabezado o cuerpo de un archivo HTML. Muchos de los ejemplos en el sitio web de Netscape, así como en otros lugares, tienden a incluir el contenedor SCRIPT en el encabezado del archivo HTML, y este es el formato preferido.

Aún así, otros desarrolladores prefieren incluir el programa JavaScript junto al elemento o sección del HTML al que se refiere, como un formulario. Dado que un archivo HTML puede contener más de una etiqueta SCRIPT, es posible colocar funciones JavaScript en lugares lógicos dentro del archivo para facilitar la codificación y depuración.

Uso de archivos externos para programas de JavaScript

Aunque incluir programas de JavaScript directamente en archivos HTML puede ser conveniente para scripts pequeños y páginas HTML básicas, puede volverse complicado cuando las páginas requieren scripts largos y complejos.

Para facilitar el desarrollo y mantenimiento de archivos HTML y scripts de JavaScript, la especificación de JavaScript incluye la opción de mantener tus scripts de JavaScript en archivos separados y utilizar el atributo src de la etiqueta SCRIPT para incluir el programa de JavaScript en un archivo HTML.

En su forma más simple, la construcción SRC se puede usar de la siguiente manera:

    
      <SCRIPT LANGUAGE="JavaScript" SRC="http://www.you.com/JavaScript.js">
      </SCRIPT>
Nota: Para que el atributo src funcione, el nombre de los archivos fuente de JavaScript debe incluir la extensión .js. Además, tu servidor web necesita estar configurado con el tipo de archivo correcto para archivos JavaScript. La información sobre esto está disponible en el sitio web de Netscape.

Uno de los beneficios de este enfoque es que tus scripts están automáticamente ocultos de otros navegadores que no soportan JavaScript. Al mismo tiempo, esta técnica requiere una solicitud adicional al servidor y acceso al servidor, lo cual puede ser problemático en un servidor lento o a través de una conexión lenta a Internet.

Además, ambas técnicas (código JavaScript en un archivo HTML y código JavaScript en un archivo externo) se pueden usar al mismo tiempo. Puedes hacer esto con un solo contenedor SCRIPT o más de uno:

      // DEJAR DE OCULTAR DE OTROS NAVEGADORES -->
      </SCRIPT>

o

Sintaxis Básica de Comandos

La sintaxis básica y estructura de JavaScript se ve familiar para cualquiera que haya utilizado C, C++, o Java. Un programa en JavaScript se construye con funciones (abordadas en el Capítulo 4) y declaraciones, operadores y expresiones. La unidad básica de comando es un comando o expresión de una línea seguido opcionalmente por un punto y coma; por ejemplo:

    
      document.writeln("¡Funciona!<BR>");

Este comando invoca el método writeln(), que es parte del objeto document. El punto y coma indica el final del comando.

Nota: Los comandos de JavaScript pueden abarcar varias líneas. Múltiples comandos también pueden aparecer en una sola línea, siempre y cuando haya un punto y coma para marcar el final de cada comando. De hecho, excepto donde sea esencial distinguir entre comandos, el punto y coma es opcional. Lo uso a lo largo de este libro para hacer el código fuente más claro y fácil de leer.

Bloques de Comandos

Múltiples comandos pueden combinarse en bloques de comandos usando llaves ({ y }). Los bloques de comandos se utilizan para agrupar conjuntos de comandos de JavaScript en una unidad única, que luego puede utilizarse para una variedad de propósitos, incluyendo bucles y definición de funciones. (Estos temas se discutirán en capítulos posteriores.) Un bloque de comandos simple se ve así:

    
      document.writeln("¿Funciona?");
      document.writeln("¡Funciona!<BR>");

Los bloques de comandos pueden ser incrustados, como ilustran las siguientes líneas.

    
      {
        código JavaScript
        {
            más código JavaScript
        }
      }

Cuando incrustas bloques de comandos de esta manera, es importante recordar que todas las llaves abiertas deben cerrarse y que la primera llave de cierre cierra la última llave abierta. En el siguiente ejemplo, la primera } cierra la segunda { como se muestra con los marcadores |:

    
      {
       |
       |  código JavaScript
       |
       |  {
       |   |   
       |   |  más código JavaScript
       |   |
       |  }
       |
       |
      }
Consejo: Al incrustar bloques de comandos unos dentro de otros, es práctica común indentar cada bloque de comandos sucesivo para que sea fácil identificar dónde comienzan y terminan los bloques al leer el código del programa. Los espacios y tabulaciones adicionales no tienen ningún efecto en los programas JavaScript.

En JavaScript, los nombres de objetos, propiedades y métodos son sensibles a mayúsculas y minúsculas, al igual que todas las palabras clave, operadores y nombres de variables. Aprenderás sobre operadores y variables en el Capítulo 3, "Trabajando con Datos e Información". De esta manera, todos los siguientes comandos son diferentes (y algunos son ilegales):

Mostrar Texto

En la mayoría de los lenguajes de programación, una de las capacidades básicas es mostrar—o desplegar—texto. En JavaScript, la salida se puede dirigir a varios lugares, incluyendo la ventana del documento actual y las cajas de diálogo emergentes.

Salida en la Ventana del Cliente

En JavaScript, los programadores pueden dirigir la salida a la ventana del cliente en secuencia con el archivo HTML. Como se discutió en la sección anterior, el JavaScript que produce salida se evalúa donde ocurre en el archivo HTML, y el texto resultante se interpreta como HTML con el propósito de mostrar la página.

Además de esto, JavaScript permite a los programadores generar cuadros de alerta y confirmación que incluyen texto y uno o dos botones. También se pueden mostrar texto y números en campos de texto (TEXT) y áreas de texto (TEXTAREA) en un formulario.

En las siguientes secciones, se revisa cómo mostrar texto en la ventana del documento.

Los Métodos document.write() y document.writeln()

El objeto document en JavaScript incluye dos métodos diseñados para mostrar texto en la ventana del cliente: write() y writeln(). En JavaScript, los métodos se llaman combinando el nombre del objeto con el nombre del método:

    
      nombre-objeto.nombre-método

Los datos que el método necesita para realizar su trabajo se proporcionan como un argumento entre paréntesis, por ejemplo:

    
      document.write("Prueba");
      document.writeln('Prueba');
Nuevo Término: Los argumentos son datos proporcionados a una función o método para su uso en cálculos y procesamiento. Se proporcionan (o pasan) a la función o método listándolos entre paréntesis después del nombre de la función o método. Múltiples argumentos se separan por comas.
Nota: Una mirada rápida a estos ejemplos muestra que las cadenas de texto están rodeadas por comillas dobles (o simples) y que los dos métodos (document.write() y document.writeln()) se invocan de la misma manera. Las comillas de apertura y cierre deben ser del mismo tipo—no se puede abrir con comillas dobles y cerrar con comillas simples o viceversa.

Trabajando con cuadros de diálogo

JavaScript proporciona la capacidad para que los programadores generen salidas en pequeños cuadros de diálogo; el contenido del cuadro de diálogo es independiente de la página HTML que contiene el script de JavaScript y no afecta la apariencia o el contenido de la página.

La forma más sencilla de dirigir una salida a un cuadro de diálogo es usar el método alert(). Para usar el método alert(), solo necesitas proporcionar una cadena de texto, como hiciste con document.write() y document.writeln() en la sección anterior:

    
      alert("Haz clic en OK para continuar.");
Nota: Notarás que el método alert() no tiene un nombre de objeto delante de él. Esto se debe a que el método alert() es parte del objeto window. Como el objeto de nivel superior en la jerarquía de objetos de Navigator, se asume el objeto window cuando no se especifica.

Generalmente, el método alert() se utiliza exactamente para eso: advertir al usuario o alertarlo sobre algo. Ejemplos de este tipo de uso incluyen:

  • Información incorrecta en un formulario
  • Un resultado inválido de un cálculo
  • Una advertencia de que un servicio no está disponible en una fecha determinada

Sin embargo, el método alert() aún puede ser utilizado para mensajes más amigables.

Nota: Ten en cuenta que los cuadros de alerta de JavaScript incluyen la frase "JavaScript Alert" al inicio del mensaje. Todos los cuadros de diálogo generados por scripts tienen encabezados similares para distinguirlos de los generados por el sistema operativo o el navegador. Esto se hace por razones de seguridad para que programas maliciosos no puedan engañar a los usuarios para que hagan cosas que no desean hacer. Los diseñadores de Netscape han indicado que pueden hacer que los cuadros de diálogo generados por scripts sean visualmente diferentes en futuras versiones.

Interactuando con el Usuario

El método alert() todavía no te permite interactuar con el usuario. La adición del botón OK te proporciona cierto control sobre el tiempo de los eventos, pero aún no se puede usar para generar ninguna salida dinámica ni personalizar la salida basada en la entrada del usuario.

La forma más simple de interactuar con el usuario es con el método prompt(). Al igual que alert(), prompt() crea un cuadro de diálogo con el mensaje que especifiques, pero también proporciona un campo de entrada único con una entrada predeterminada. El usuario necesita completar el campo y luego hacer clic en OK. Un ejemplo del método prompt() es la siguiente línea, que genera un cuadro de diálogo como el de la Figura 2.11:

    
      prompt("Enter Your favourite color:", "Blue");

Inmediatamente notarás una diferencia con la forma en que usaste el método alert(): estás proporcionando dos cadenas de texto al método entre paréntesis. El método prompt() requiere dos piezas de información: la primera es el texto que se mostrará, y la segunda es el dato predeterminado en el campo de entrada.

Nota: Las piezas de información proporcionadas entre paréntesis a un método o función se conocen como argumentos. En JavaScript, cuando un método requiere más de un argumento, estos se separan por comas.

Puede que hayas notado que si se usa por sí solo, el método prompt() acepta la entrada del usuario, pero la información se pierde esencialmente. Esto se soluciona al darse cuenta de que los métodos y funciones pueden devolver resultados, como se mencionó en el capítulo anterior. Eso significa que el método prompt() devolverá la entrada del usuario como su resultado.

El resultado devuelto por un método puede almacenarse en una variable (lo cual aprenderás en el próximo capítulo) o puede usarse como un argumento para otro método:

    
      document.write("Tu color favorito es: ");
      document.writeln(prompt("Introduce tu color favorito:", "Azul"));

En la segunda línea de este segmento de código, el método document.writeln() muestra los resultados devueltos por el método prompt(), como se ilustra en las Figuras 2.12 y 2.13.

Nuevo Término: La concatenación se discute con más detalle en el Capítulo 3. Usando la concatenación, múltiples cadenas se combinan en una sola cadena y son tratadas como una sola cadena por JavaScript.

Para hacer esto, puedes combinar las diversas piezas de tu mensaje de bienvenida en un solo comando document.write() utilizando un simple signo más (+):

Tipos de Datos en JavaScript

JavaScript utiliza cuatro tipos de datos: números, cadenas, valores booleanos y un valor nulo para representar toda la información que el lenguaje puede manejar. En comparación con la mayoría de los lenguajes, este es un número pequeño de tipos de datos, pero es suficiente para manejar inteligentemente la mayoría de los datos utilizados en todo excepto los programas más complejos.

Los cuatro tipos de datos en JavaScript se describen en la Tabla 3.1.

Tabla 3.1. Tipos de datos en JavaScript.
Tipo Ejemplo
Números Cualquier número, como 17, 21.5, o 54e7
Cadenas "¡Saludos!" o '¡Diversión!'
Booleano Verdadero o falso
Nulo Una palabra clave especial para exactamente eso: el valor null (es decir, nada)

Literales

El término literales se refiere a la manera en que se representan cada uno de los cuatro tipos de datos. Los literales son valores fijos que proporcionan literalmente un valor en un programa. Por ejemplo, 11 es un número literal, "hola" es una cadena literal, y true es un literal booleano.

Ya has visto literales en uso en los capítulos anteriores cuando diste argumentos a diferentes métodos en forma de cadenas de texto como "¡Bienvenido a Netscape Navigator 2.0!" y "Ingrese su nombre:".

Para cada tipo de dato, hay diferentes formas de especificar literales.

Números

El tipo de número en JavaScript abarca lo que serían varios tipos en lenguajes como Java. Usando los números, es posible expresar tanto enteros como valores de punto flotante.

Enteros

Los enteros son números sin ninguna porción que siga al punto decimal; es decir, son números enteros, sin fracciones. Los enteros pueden ser números positivos o negativos. El tamaño máximo de un entero depende de la plataforma que ejecute la aplicación JavaScript.

En JavaScript, puedes expresar enteros en tres bases diferentes: base 10 (decimal, lo que normalmente usas en situaciones cotidianas), base 8 (conocida como octal) y base 16 (hexadecimal).

Nota: Los números en base 8 solo pueden tener dígitos hasta el 7, por lo que un valor decimal de 18 sería un valor octal de 22. De manera similar, el sistema hexadecimal permite dígitos hasta la F, donde A equivale a 10 en decimal y F es 15. Entonces, un valor decimal de 18 sería 12 en notación hexadecimal.

Para distinguir entre estas tres bases, JavaScript usa las notaciones descritas en la Tabla 3.2 para especificar las diferentes bases.

Tabla 3.2. Especificación de bases en JavaScript.
Sistema Numérico Notación
Decimal (base 10) Un entero normal sin un 0 (cero) al inicio (por ejemplo, 752)
Octal (base 8) Un entero con un 0 (cero) al inicio (por ejemplo, 056)
Hexadecimal (base 16) Un entero con un Ox o OX al inicio (por ejemplo, Ox5F o OXC72)
Valores de Punto Flotante

Los valores de punto flotante pueden incluir una componente fraccionaria. Un literal de punto flotante incluye un número entero decimal más un punto decimal y una fracción expresada como otro número decimal o un indicador de exponente y un sufijo de tipo, como se muestra en los siguientes ejemplos:

  • 7.2945
  • -34.2
  • 2E3

Los literales de punto flotante deben, como mínimo, incluir un número entero decimal y ya sea el punto decimal o el indicador de exponente (“e” o “E”). Al igual que los enteros, los valores de punto flotante pueden ser positivos o negativos.

Nota: Debe tenerse en cuenta que el manejo de números de punto flotante por parte de JavaScript puede introducir inexactitudes en algunos cálculos. Debes tener esto en cuenta para tus programas.

Cadenas (Strings)

Ya has encontrado literales de cadena en el Capítulo 2, “Tu Primer Script”, donde los utilizaste como argumentos para varios métodos.

Técnicamente, un literal de cadena contiene cero o más caracteres encerrados, como sabes, entre comillas simples o dobles:

  • "Hola!"
  • '245'
  • " "
Nuevo Término: El último ejemplo se llama la cadena vacía. Es importante notar que la cadena vacía es distinta del valor null en JavaScript.
Nota: Las cadenas son diferentes de otros tipos de datos en JavaScript. Las cadenas son en realidad objetos (los objetos se presentan en el Capítulo 4).

Booleano

Un literal booleano puede tomar dos valores: true o false. Este tipo de literal es útil al comparar datos y tomar decisiones, como verás más adelante en este capítulo.

Nota: A diferencia de Java, C y otros lenguajes, en JavaScript los valores booleanos solo pueden representarse con true y false. Los valores de 1 y @ no se consideran valores booleanos en JavaScript.

El Valor null

El valor null es un valor especial en JavaScript. El valor null representa justamente eso: nada. Si intentas referenciar una variable que no está definida y, por lo tanto, no tiene valor, el valor devuelto es null. Asimismo, en un cuadro de diálogo de solicitud, si el usuario selecciona el botón Cancelar, se devuelve un valor null.

Esto es distinto de un valor de cero o una cadena vacía donde hay un valor real.

El valor null se indica en JavaScript con el término null.

NaN

Además de estos valores, algunas funciones devuelven un valor especial llamado NaN, que significa que el valor no es un número. parseInt() y parseFloat() en sistemas UNIX son ejemplos de funciones que devuelven NaN cuando el argumento pasado a ellas no puede evaluarse como un número.

Los valores pueden ser probados para ver si son NaN usando la función isNaN(), la cual devuelve true o false basado en la naturaleza del argumento pasado a la función.

Conversión de tipos (Casting)

JavaScript es lo que se llama un lenguaje de programación de tipado débil. En los lenguajes de tipado débil, el tipo de un literal o variable (que discutimos en la siguiente sección) no se define cuando se crea una variable y puede, a veces, cambiar según el contexto. En comparación, Java y C no son de tipado débil.

En sus formas más tempranas, LiveScript y JavaScript permitían a los programadores combinar dos literales de diferentes tipos, resultando en un valor literal del mismo tipo que el primer literal en la expresión. Por ejemplo, combinar la cadena "Contar hasta " con el literal entero 10 resulta en una cadena con el valor "Contar hasta 10".

Por el contrario, sumar el literal numérico 3.5 y la cadena "10" resulta en el literal numérico de punto flotante 13.5.

Nuevo Término: Este proceso se conoce como conversión de tipos (casting). El primer ejemplo convierte el número 10 en una cadena, y el segundo convierte la cadena "10" en un número.

Sin embargo, a medida que JavaScript y Java se han acercado más, esto ha comenzado a cambiar. En la versión de JavaScript actualmente disponible, ya no es posible convertir una cadena en un número usando una forma como 0 + "1". JavaScript ha añadido las funciones parseInt() y parseFloat(), que convierten cadenas en números enteros o de punto flotante. Por ejemplo, parseInt("13") devuelve el entero 13 y parseFloat("45.2") devuelve el número de punto flotante 45.2.

Todavía es posible convertir un número en una cadena como en "Contar hasta " + 10 evaluando a una cadena con el valor "Contar hasta 10".

Creación de Variables

Para hacer que el trabajo con tipos de datos sea útil, necesitas maneras de almacenar valores para su uso posterior. Aquí es donde entran las variables.

En JavaScript puedes crear variables que pueden contener cualquier tipo de dato. Las variables tienen nombres, y después de asignarles valores, puedes referirte al valor por su nombre. Si posteriormente asignas un nuevo valor a la variable, puedes seguir refiriéndote a ese nuevo valor por el nombre de la variable.

Declaración de Variables

Para utilizar una variable, es una buena práctica de programación declararla. Declarar una variable le indica a JavaScript que existe una variable con un nombre determinado para que el intérprete de JavaScript pueda entender las referencias a ese nombre de variable en todo el resto del código.

Si bien es posible declarar variables simplemente usándolas, declarar variables ayuda a garantizar que los programas estén bien organizados y a realizar un seguimiento del alcance de las variables (se analiza en el Capítulo 4, "Funciones y Objetos: los componentes básicos de los programas").

Puedes declarar una variable usando el comando var:

  
    var ejemplo;

En esta línea, has definido una variable llamada ejemplo, que actualmente no tiene ningún valor. También es posible asignar un valor a una variable cuando la declaras:

  
    var ejemplo = "Un Ejemplo";

Aquí has declarado la variable llamada ejemplo y le has asignado un valor de cadena de texto "Un Ejemplo". Debido a que JavaScript también permite declarar variables en el primer uso, el comando ejemplo = "Un Ejemplo" también lograría el mismo resultado.

Nuevo término: El signo igual (=) que se usa para asignar un valor a una variable se conoce como operador de asignación. Los operadores de asignación se analizan más adelante en este capítulo.

Para comprender mejor cómo declarar, asignar y usar variables, el siguiente segmento de código produce una salida similar a la Figura 3.1.

  
    var example="Un Ejemplo";
    document.write(example) ;

Nombres de Variables Válidos

Al igual que los nombres de propiedades y métodos en JavaScript, los nombres de variables son sensibles a mayúsculas y minúsculas. Además, los nombres de variables deben comenzar con una letra o un guion bajo (_). Después de eso, los caracteres restantes también pueden incluir números.

Trabajar con Variables—Expresiones

Para que las variables sean útiles, debes poder manipular variables y evaluarlas en diferentes contextos.

Esta capacidad es proporcionada por las expresiones. En su forma más básica, una expresión no es más que una colección de variables, operadores y otras expresiones, todas las cuales se evalúan a un solo valor.

En términos prácticos, eso significa que hay dos tipos de expresiones: aquellas que simplemente tienen un valor y aquellas que asignan un valor a una variable. Has visto ejemplos simples de ambos: ejemplo = "Un Ejemplo" es una expresión que asigna un valor a la variable ejemplo, mientras que "El Número es " + "10" es un ejemplo de una expresión que simplemente tiene un valor.

Al igual que con los tipos de datos, JavaScript tiene varios tipos de expresiones:

  • Asignación: Asigna un valor a una variable
  • Aritmética: Evalúa a un número
  • Cadena: Evalúa a una cadena
  • Lógica: Evalúa a un valor booleano

Expresiones de Asignación

Las expresiones de asignación utilizan operadores de asignación para asignar un valor a una variable. La estructura típica de una expresión de asignación es:

  variable operador expresión

En otras palabras, el operador asigna un valor a la variable mediante la realización de algún tipo de operación sobre la expresión. La Tabla 3.3 describe los operadores de asignación en JavaScript.

Nuevo Término :Técnicamente, el elemento a la izquierda del operador se llama operando izquierdo y el elemento a la derecha se llama operando derecho.
Tabla 3.3. Operadores de asignación en JavaScript.
Operador Descripción
= Asigna el valor del operando derecho al operando izquierdo
+= Suma los operandos izquierdo y derecho y asigna el resultado al operando izquierdo
-= Resta el operando derecho del operando izquierdo y asigna el resultado al operando izquierdo.
*= Multiplica los dos operandos y asigna el resultado al operando izquierdo
/= Divide el operando izquierdo por el operando derecho y asigna el valor al operando izquierdo
%= Divide el operando izquierdo por el operando derecho y asigna el resto al operando izquierdo
Nota: El operando %= asigna el módulo al operando de la izquierda. Es decir, x %= y es lo mismo que x = x % y. El módulo es el residuo cuando dos números se dividen.

Por ejemplo, si x = 10 e y = 5, entonces x += y establece x en 15, x *= y establece x en 50, x /= y establece x en 2, y x %= y establece x en cero porque el residuo de 10 dividido por 5 es cero.

Nota: Existen otros operadores de asignación conocidos como operadores de asignación bit a bit, como <<=, >>=, y ^=, pero estos son avanzados y requieren un entendimiento de números binarios (base 2).

Otros operadores:

Además de los operadores de asignación que ya hemos discutido, JavaScript también tiene operadores para expresiones que simplemente se evalúan en un valor. Estos son los operadores aritméticos, operadores de cadena y operadores lógicos, así como los operadores bit a bit, que están fuera del alcance de este libro.

Como verás en los siguientes ejemplos, estos operadores incluyen tanto operadores que requieren dos operandos como aquellos que requieren un solo operando.

Nuevo término: Un operador que requiere un solo operando se conoce como operador unario, y uno que requiere dos operandos es un operador binario.

Además, todos los operadores discutidos aquí pueden tomar expresiones como sus operandos. En estos casos, las expresiones que actúan como operandos se evalúan antes de evaluar la expresión final. Por ejemplo, en la siguiente sección, aprenderás sobre expresiones aritméticas simples como 15 + 3. Dado que los operadores pueden tomar otras expresiones como sus operandos, podrías escribir una expresión como x += 15 + 3. Esto suma 15 y 3, luego suma el resultado (18) al valor de x y asigna el resultado final a x.

Operadores aritméticos

Los operadores aritméticos binarios estándar son los mismos que los de una calculadora básica: suma (+), resta (-), multiplicación (*) y división (/). Además de estos operadores básicos está el operador de módulo (%), que, como se mencionó antes, calcula el residuo después de dividir sus operandos. Los siguientes son ejemplos de expresiones válidas utilizando estos:

  
    8 + 5
    32.5 - 72.3
    12 % 5

Además de estos operadores binarios, hay tres operadores aritméticos unarios que son bastante útiles: incremento (++), decremento (--) y negación unaria (-).

Tanto el operador de incremento como el de decremento pueden ser utilizados de dos maneras diferentes: antes del operando o después. Por ejemplo, ++x incrementa x en uno y devuelve el resultado, mientras que x++ devuelve x y luego incrementa el valor de x. De manera similar, --x decrementa el valor de x en uno antes de devolver el resultado, mientras que x-- devuelve el valor de x antes de disminuir su valor en uno.

Por ejemplo:

  
    x = 5;
    y = ++x;
    z = x++;

En estas líneas de código, primero se asigna a x el valor de 5, luego se incrementa a 6 y se asigna a y. Después, el nuevo valor de 6 se asigna a z y el valor de x se incrementa a 7. Así que, al final, x es 7, y es 6, y z es 6.

La negación unaria funciona de manera un poco diferente. El operador debe preceder a su único operando, y el resultado es la negación del operando. Típicamente, el uso de este operando se ve así:

  
    x = -x;

Aquí, si el valor de x es 5, entonces se convierte en -5. Asimismo, si x fuera -4, se convertiría en 4.

Operadores Lógicos

Los operadores lógicos incluyen tanto operadores binarios como unarios. Toman valores booleanos como operandos y devuelven valores booleanos, como se detalla en la Tabla 3.4.

Tabla 3.4. Operadores lógicos en JavaScript.
Operador Descripción
&& Lógico "y" —devuelve true cuando ambos operandos son true; de lo contrario, devuelve false.
|| Lógico "o" —devuelve true si alguno de los operandos es true. Solo devuelve false cuando ambos operandos son false.
! Lógico "no" —devuelve true si el operando es false y false si el operando es true. Este es un operador unario y precede al operando.

Al discutir operadores y expresiones lógicas, es necesario hablar de la evaluación de cortocircuito. Con la evaluación de cortocircuito, JavaScript terminará de evaluar una expresión después de evaluar el primer operando (izquierdo), si el primer operando proporciona suficiente información para evaluar la expresión. La evaluación de cortocircuito utiliza las siguientes reglas:

  • false && cualquier cosa siempre es false.
  • true || cualquier cosa siempre es true.

Por ejemplo, si x es igual a 10 y y es igual a 20, entonces la expresión (x > y) && (x < y) se evaluaría inmediatamente como false una vez que la primera parte de la expresión (x > y) se evalúe como false. De igual manera, (y > x) || (x > y) se evalúa como true simplemente porque la primera parte de la expresión (y > x) es true. Estos ejemplos utilizan operadores de comparación, los cuales se discuten en la siguiente sección.

Debido a que el operador lógico "no" (!) toma un solo operador, no hay evaluación de cortocircuito para él.

Operadores de Comparación

Los operadores de comparación son similares a los operadores lógicos en que devuelven valores booleanos. A diferencia de los operadores lógicos, no requieren que sus operandos sean valores booleanos.

Los operadores de comparación se utilizan para comparar el valor de los operandos por igualdad, así como por una serie de otras condiciones. La Tabla 3.5 enumera los operadores de comparación disponibles en JavaScript.

Tabla 3.5. Operadores de comparación en JavaScript.
Operador Descripción
== Devuelve true si los operandos son iguales
!= Devuelve true si los operandos no son iguales
> Devuelve true si el operando izquierdo es mayor que el operando derecho
< Devuelve true si el operando izquierdo es menor que el operando derecho
>= Devuelve true si el operando izquierdo es mayor o igual que el operando derecho
<= Devuelve true si el operando izquierdo es menor o igual que el operando derecho

En JavaScript, todos los operadores de comparación son operadores binarios.

Los operadores de comparación se pueden usar para comparar números, así como cadenas de texto; por ejemplo:

1 == 1 devuelve verdadero.
3 < 1 devuelve falso.
5 >= 4 devuelve verdadero.
"the" != "he" devuelve verdadero.
4 == "4" devuelve verdadero.
Nota: Al comparar valores de cadena y numéricos, si el valor de la cadena comienza con caracteres no numéricos, JavaScript generará un error.
Operadores Condicionales

Las expresiones condicionales son un poco diferentes de las otras que has visto hasta ahora porque una expresión condicional puede evaluar a uno de dos valores diferentes según una condición. La estructura de una expresión condicional es:

  
    (condición) ? val1 : val2

La forma en que funciona una expresión condicional es que la condición, que es cualquier expresión que se pueda evaluar a un valor booleano, se evalúa; según el resultado, toda la expresión se evalúa como val1 (condición verdadera) o val2 (condición falsa).

La expresión

  
    (día == "Sábado") ? "Fin de semana!" : "No es Sábado!"

se evalúa como "Fin de semana!" cuando día es "Sábado". De lo contrario, la expresión se evalúa como "No es Sábado!".

Operadores de Cadenas

En el Capítulo 2, aprendiste a usar el operador de concatenación (+). La concatenación devuelve la unión de dos cadenas de manera que

  
    "Bienvenido a" + "Netscape Navigator"

se evalúa como una sola cadena con el valor "Bienvenido a Netscape Navigator." Al igual que con los números, esto se puede hacer con un operador de concatenación abreviado. Por ejemplo, si la variable bienvenido tiene el valor "Bienvenido a", entonces

  
    bienvenido += "Netscape Navigator";

asignaría la cadena "Bienvenido a Netscape Navigator" a la variable bienvenido.

Precedencia de Operadores

Debido a que las expresiones pueden ser los operandos de otras expresiones, es necesario entender la precedencia de los operadores. La precedencia de operadores es el conjunto de reglas que determina el orden en el cual se evalúan estas expresiones compuestas.

Los operadores que has aprendido se evalúan en el siguiente orden (de menor a mayor precedencia):

Operadores de asignación (= += -= = /= %=)
Condicional (?:)
Lógico o (||)
Lógico y (&&)
Igualdad (== !=)
Relacionales (< <= > >=)
Suma/resta (+ -)
Multiplicación/división/módulo (* / %)
Paréntesis (())

Basado en estas reglas, la expresión

  
    5 + 3 * 2

se evalúa como

  
    5 + 6

lo cual se evalúa como 11. Sin estas reglas, el operador de suma se evaluaría antes que el operador de multiplicación y el resultado sería 16. De manera similar, la expresión

  
    false || true && false

se evalúa como

  
    false

porque la expresión && se evalúa primero como false, y luego la expresión || (que se convierte en false || false) se evalúa como false.

Las reglas de precedencia de operadores pueden ser anuladas mediante el uso de paréntesis. Las expresiones entre paréntesis se evalúan antes que las fuera de los paréntesis, de modo que la expresión

  
    (5 + 3) * 2

se evaluaría como 16, en lugar de 11 sin los paréntesis.

Aplicando Comparación: Construcciones if-else

Ahora que sabes cómo crear expresiones y, lo que es más importante, cómo crear expresiones de comparación y expresiones lógicas, estás listo para aplicarlas.

En la sección anterior viste cómo una expresión, como una comparación, podía ser la condición en una expresión condicional. El operador condicional te ofrece una manera simple de tomar una decisión: evaluar un valor cuando la condición es verdadera y otro cuando la condición es falsa.

Aun así, al usar expresiones condicionales, no puedes ir más allá de los límites de un flujo de programa lineal. Es decir, cada línea del script se evalúa y se ejecuta en orden; aún no puedes alterar el orden.

Usando la estructura if-else, combinada con expresiones, puedes alterar el flujo de un programa para determinar qué secciones del código se ejecutan según una condición.

En su estructura más simple, la declaración if se usa de la siguiente manera:

  
    if condición
       comando;

Es decir, si la condición es verdadera, ejecuta el comando. De lo contrario, no lo ejecutes y pasa al siguiente comando o condición. Sin embargo, como aprendiste en el Capítulo 2, los comandos pueden agruparse en bloques de comandos utilizando llaves. La declaración if también puede usarse con bloques de comandos:

  
    if condición {
       varias líneas de código JavaScript
    }

Por ejemplo, estas líneas

  
    if (día == "sábado") {
      document.writeln("¡Es el fin de semana!");
      alert("¡Es el fin de semana!");
    }

escribirán "¡Es el fin de semana!" tanto en la ventana del documento como en un cuadro de diálogo de alerta solo si la variable día tiene el valor "sábado". Si día tiene cualquier otro valor, ninguna línea se ejecuta.

Usando esto puedes tener un mensaje diferente tanto para el sábado como para cualquier otro día de la semana:

  
    if (día == "sábado") {
      document.writeln("¡Es el fin de semana!");
    }
    if (día != "sábado") {
      document.writeln("No es sábado.");
    }

La estructura if-else proporciona una manera más fácil de hacer esto usando else:

  
    if (día == "sábado") {
      document.writeln("¡Es el fin de semana!");
      } else {
      document.writeln("No es sábado.");
    }

La estructura else permite la creación de un bloque de comandos para ejecutar cuando la condición en la sentencia if asociada es falsa.

Además, ten en cuenta que las estructuras if-else pueden estar anidadas:

  
    if condición1 {
      Comandos JavaScript
      if condición2 {
        Comandos JavaScript
      } else {
        Otros comandos
      }
      Más comandos JavaScript
    } else {
      Otros comandos
    }

Usando if para Repetición

Usando la sentencia if, vas a extender el Listado 3.3 un paso más: vas a permitir que el usuario indique si quiere una segunda oportunidad para responder la pregunta correctamente, como se muestra en el Listado 3.4.

Lo que quieres hacer es formular la pregunta y verificar el resultado. Si el resultado es incorrecto, preguntarás al usuario si desea intentarlo de nuevo. Si lo desea, le preguntarás una vez más.

Para facilitar esto, usarás el método confirm(), que es similar a los métodos alert() y prompt() que ya sabes cómo usar. El método confirm() toma una única cadena como argumento. Muestra la cadena en un cuadro de diálogo con los botones OK y Cancelar y devuelve un valor de true si el usuario selecciona OK o false si selecciona Cancelar.

Entrada: Listado 3.4. El método confirm() con la sentencia if. 
  
    <HTML>
    <HEAD>
    <TITLE>Ejemplo 3.4</TITLE>
    <SCRIPT LANGUAGE="JavaScript">
    <!-- OCULTAR DE OTROS NAVEGADORES
    // DEFINIR VARIABLES PARA EL RESTO DEL SCRIPT
    var pregunta="¿Qué es 10+10?";
    var respuesta=20;
    var correcto='<IMG SRC="correcto.gif">';
    var incorrecto='<IMG SRC="incorrecto.gif">';
    // HACER LA PREGUNTA

      }

    var response = prompt(pregunta,"0");

En esta línea, declaras la variable response, pides al usuario que responda la pregunta y asignas la respuesta del usuario a response.

  
    if (response != respuesta)

Aquí, comparas la respuesta del usuario con la respuesta correcta. Si la respuesta es incorrecta, entonces se ejecuta la siguiente línea. Si la respuesta es correcta, el programa salta hacia abajo para mostrar el resultado.

  
    if (confirm("¡Incorrecto! Presiona OK para una segunda oportunidad."))

El usuario ha dado una respuesta incorrecta. Ahora verificas si el usuario quiere una segunda oportunidad con el método confirm(), que devuelve un valor booleano, el cual es evaluado por la declaración if.

  
    response = prompt(pregunta,"0");

Si el usuario selecciona OK en el cuadro de diálogo de confirmación, el método confirm() devuelve true, y esta línea se ejecuta. Con este comando, se vuelve a preguntar al usuario la pregunta, y la segunda respuesta se almacena en la variable response, reemplazando la respuesta anterior.

Resumen

JavaScript tiene cuatro tipos de datos básicos: numérico (tanto entero como de punto flotante), cadena de texto, booleano y el valor nulo. Los literales, que expresan un valor literalmente, pueden ser de tipo numérico, de cadena de texto o booleano; reglas específicas gobiernan el formato de estos literales.

Las variables, que son contenedores nombrados para almacenar datos e información en un programa, se declaran usando la declaración var. Debido a que JavaScript es un lenguaje de tipado débil, el tipo de literales y variables cambia dinámicamente dependiendo de las acciones que se realicen sobre ellos o con ellos.

Las expresiones proporcionan un medio para analizar y trabajar con variables y literales. Hay varios tipos de expresiones, incluidas las expresiones de asignación, expresiones aritméticas, expresiones lógicas, expresiones de comparación, expresiones de cadena de texto y expresiones condicionales. Las expresiones están compuestas por una serie de operandos y operadores que evalúan a un solo valor.

Las reglas de precedencia de operadores te indican el orden en el que se evaluarán las expresiones compuestas. Se pueden usar paréntesis para anular la precedencia de operadores.

La construcción if-else te permite decidir qué código del programa se ejecutará según el valor de variables, literales o expresiones.

En el Capítulo 4, veremos funciones y objetos como los bloques de construcción de la mayoría de los programas.

Capítulo 4
Funciones y Objetos — Los Componentes Fundamentales de los Programas

Una vez que comiences a escribir programas más complejos, rápidamente te darás cuenta de la necesidad de realizar algunas tareas y acciones más de una vez durante el transcurso de un programa.

Esta necesidad se aborda con las funciones, que son similares a los métodos pero no están adjuntas a ningún objeto en particular. Como programador, puedes crear numerosas funciones en tus programas—esto ayuda a organizar la estructura de tus aplicaciones y facilita el mantenimiento y la modificación de tu código de programa.

Además, las funciones son particularmente útiles para trabajar con eventos y manejadores de eventos, como aprenderás en el Capítulo 5, "Eventos en JavaScript".

También puedes usar funciones como base para crear tus propios objetos y complementar aquellos que están disponibles en JavaScript.

En este capítulo cubriremos los siguientes temas:

  • La naturaleza de las funciones
  • Funciones integradas versus funciones creadas por el programador
  • Cómo definir y usar funciones
  • Cómo crear nuevos objetos, propiedades y métodos
  • Cómo usar arreglos asociativos

¿Qué son las funciones?

Las funciones ofrecen a los programadores la capacidad de agrupar código de programa que realiza una tarea específica—o función—en una sola unidad que se puede usar repetidamente a lo largo de un programa.

Al igual que los métodos que has visto en capítulos anteriores, una función se define por su nombre y se invoca usando su nombre.

También, al igual que algunos de los métodos que has visto antes (como prompt() y confirm()), las funciones pueden aceptar información en forma de argumentos y pueden devolver resultados.

JavaScript incluye varias funciones incorporadas, así como métodos de objetos base. Ya has visto esto cuando usaste alert(), document.write(), parseInt(), o cualquiera de los otros métodos y funciones con los que has estado trabajando. Sin embargo, la flexibilidad de JavaScript radica en la capacidad de los programadores para crear sus propias funciones para complementar las disponibles en la especificación de JavaScript.

Usando funciones

Para poder utilizar las funciones, necesitas saber cómo definirlas, pasar argumentos a ellas y devolver los resultados de sus cálculos. También es importante entender el concepto de ámbito de las variables, que rige si una variable está disponible en todo un script o solo dentro de una función específica.

Definición de Funciones

Las funciones se definen utilizando la declaración de función. La declaración de función requiere un nombre para la función, una lista de parámetros—o argumentos—que se pasarán a la función, y un bloque de comandos que define lo que la función hace:

  
    function nombre_de_la_función(parámetros, argumentos) {
      bloque de comandos
    }

Como notarás, la nomenclatura de las funciones sigue básicamente las mismas reglas que las variables: son sensibles a mayúsculas y minúsculas, pueden incluir guiones bajos (_), y comienzan con una letra. La lista de argumentos que se pasan a la función aparece entre paréntesis y se separa por comas.

Es importante darse cuenta de que definir una función no ejecuta los comandos que la componen. Solo cuando la función es llamada por su nombre en alguna otra parte del script es que la función se ejecuta.

Pasando Parámetros

En la siguiente función, puedes ver que printName() acepta un argumento llamado name:

  
    function printName(name) {
      document.write("<HR>Tu Nombre es <B><I>");
      document.write(name);
      document.write("</B></I><HR>");
    }

Dentro de la función, las referencias a name se refieren al valor pasado a la función.

Hay varios puntos a tener en cuenta aquí:

  • Tanto las variables como los literales se pueden pasar como argumentos al llamar a una función.
  • Si se pasa una variable a la función, cambiar el valor del parámetro dentro de la función no cambia el valor de la variable pasada a la función.
  • Los parámetros existen solo durante la vida de la función—si llamas a la función varias veces, los parámetros se crean de nuevo cada vez que llamas a la función, y los valores que tenían cuando la función terminó la última vez no se retienen.

Por ejemplo, si llamas a printName() con el comando:

  
    printName("Bob");

Entonces, cuando printName() se ejecute, el valor de name será "Bob". Si llamas a printName() usando una variable como argumento:

  
    var user = "John";
    printName(user);

entonces el nombre tiene el valor "John". Si añadieras una línea a printName() cambiando el valor de nombre:

  
    name = "Mr. " + name;

el nombre cambiaría, pero la variable usuario, que se envió como argumento, no cambiaría.

Nota: Al pasar argumentos a una función, se crean dos propiedades que pueden ser útiles al trabajar con los argumentos: functionname.arguments y functionname.arguments.length.
functionname.arguments es un array con una entrada para cada argumento y functionname.arguments.length es una variable entera que indica el número de variables pasadas a la función. Puedes usar estas propiedades para producir funciones que acepten un número variable de argumentos.

Alcance de las Variables (Variable Scope)

Esto nos lleva a una discusión sobre el alcance de las variables. El alcance de las variables se refiere a dónde existe una variable.

Por ejemplo, en el ejemplo printName(), el nombre solo existe dentro de la función printName()—no se puede referir ni manipular fuera de la función. Entra en existencia cuando se llama a la función y deja de existir cuando la función termina. Si la función se llama nuevamente, se crea una nueva instancia de nombre.

Además, cualquier variable declarada usando el comando var dentro de la función tendrá un alcance limitado a la función.

Si una variable se declara fuera del cuerpo de una función, está disponible en todo el script—dentro de todas las funciones y en otros lugares.

Nuevo Término: Las variables declaradas dentro de una función se conocen como variables locales. Las variables declaradas fuera de las funciones y disponibles en todo el script se conocen como variables globales.

Si declaras una variable local dentro de una función que tiene el mismo nombre que una variable global existente, entonces dentro de la función, ese nombre de variable se refiere a la nueva variable local y no a la variable global. Si cambias el valor de la variable dentro de la función, no afecta el valor de la variable global.

Retornando Resultados

Como se mencionó en la sección anterior, las funciones pueden retornar resultados. Los resultados se retornan utilizando la declaración return. La declaración return puede utilizarse para devolver cualquier expresión válida que se evalúe a un único valor. Por ejemplo, en la función cube(),

  
    function cube(number) {
      var cube = number * number * number;
      return cube;
    }

La declaración de retorno devolverá el valor de la variable cubo. Esta función podría haber sido escrita de la siguiente manera con la misma facilidad: 

  
    function cube(number) {
      return number * number * number;
    }

Esto funciona porque la expresión number * number * number se evalúa a un único valor.

Funciones en el encabezado del archivo

Como se mencionó en el Capítulo 3, "Trabajando con Datos e Información", existen razones convincentes para incluir las definiciones de funciones dentro de las etiquetas HEAD del archivo HTML.

Esto asegura que todas las funciones hayan sido interpretadas antes de que sea posible que los eventos del usuario invoquen una función. Esto es especialmente relevante cuando se empieza a trabajar con manejadores de eventos, donde la ubicación incorrecta de una definición de función puede significar que un evento pueda provocar la llamada a una función que aún no ha sido evaluada y Navigator no sabe que existe. Cuando esto ocurre, se muestra un mensaje de error.

Nuevo término: El término "parsed" se refiere al proceso mediante el cual el intérprete de JavaScript evalúa cada línea de código de script y la convierte en un Byte Code pseudo-compilado (similar a Java), antes de intentar ejecutarlo. En este momento, se pueden detectar y reportar errores de sintaxis y otros errores de programación que podrían evitar que el script se ejecute, informándolos al usuario o al programador.

El operador typeof

JavaScript ofrece un operador que no discutimos en el último capítulo cuando examinamos variables, expresiones y operadores. El operador `typeof` se utiliza para identificar el tipo de un elemento en JavaScript. Dado un operando no evaluado (como un nombre de variable o un nombre de función), el operador `typeof` devuelve una cadena que identifica el tipo del operando.

Por ejemplo, supongamos que tienes el siguiente código JavaScript: 

  
    var pregunta="What is 10 x 10?";
    var respuesta=10;
    var correcto=false;
    function showResult(results) {
    document.write(results) ;
    }

Entonces, typeof pregunta devuelve cadena; typeof respuesta devuelve número; typeof correcto devuelve booleano; y typeof showResult devuelve función. Otros resultados posibles devueltos por el operador typeof incluyen undefined y objeto.

Nota: El operador typeof puede ser útil para determinar si una función ha sido cargada y está lista para ser llamada. Esto es especialmente útil en páginas con múltiples marcos (frames) donde un script cargado en un marco (frame) podría intentar llamar a una función ubicada en otro marco (frame) que aún no ha completado su carga.

Los marcos (frames) se tratan con más detalle en el Capítulo 8.

Funciones Recursivas

Ahora que has visto un ejemplo de cómo funcionan las funciones, echemos un vistazo a una aplicación de las funciones llamada recursión.

Nuevo Término: La recursión se refiere a situaciones en las que las funciones se llaman a sí mismas. Este tipo de funciones se conocen como funciones recursivas.

Por ejemplo, lo siguiente es un ejemplo de una función recursiva que calcula un factorial:

Nota: Un factorial es una función matemática. Por ejemplo, el factorial de 5 (escrito 5!) es igual a 5x4x3x2x1 y 7! = 7x6x5x4x3x2x1. 
  
    function factorial(number) {
      if (number > 1) {
      return number * factorial(number - 1);
      } else {
      return number;
      }
    }

A primera vista, esta función puede parecer extraña. Esta función se basa en el hecho de que el factorial de un número es igual al número multiplicado por el factorial del número menos uno. Expresado matemáticamente, esto podría escribirse así: 

  
    x! = x * (x-1)!

Para aplicar esta fórmula, has creado una función recursiva llamada factorial(). La función recibe un número como argumento. Utilizando la siguiente estructura if-else: 

  
    if (number > 1) {
      return number * factorial(number - 1);
      } else {
      return number;
    }

La función devuelve un valor de 1 si el argumento es igual a 1, o aplica la fórmula y devuelve el número multiplicado por el factorial del número menos uno.

Para lograr esto, debe llamar a la función factorial() desde dentro de la función factorial(). Aquí es donde el concepto de alcance de variables se vuelve extremadamente importante. Es crucial darse cuenta de que cuando la función llama a factorial(), se invoca una nueva instancia de la función, lo que significa que se crea una nueva instancia de número. Esto continúa ocurriendo hasta que la expresión number - 1 tenga un valor de 1.

Consejo: Las funciones recursivas son poderosas, pero pueden ser peligrosas si no se tiene cuidado con la recursión infinita. La recursión infinita ocurre cuando la función está diseñada de tal manera que se llama a sí misma indefinidamente sin detenerse.
A nivel práctico, en JavaScript, la recursión infinita no es probable que ocurra debido a la forma en que JavaScript maneja la asignación de memoria. Esto significa que las recursiones profundas, incluso si no son infinitas, pueden hacer que el navegador se bloquee.

Es importante destacar que la función factorial() previene la recursión infinita porque la estructura if-else asegura que eventualmente la función dejará de llamarse a sí misma una vez que el número pasado a ella sea igual a uno. Además, si la función es llamada inicialmente con un valor menor que dos, la función retornará inmediatamente sin recursión alguna.
